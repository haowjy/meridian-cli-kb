# meridian-web Vocabulary

Terms covering the extension system, shell architecture, WebSocket protocol, layout, and frontend stores. For the full vocabulary index see [../../vocabulary.md](../../vocabulary.md).

| Term | Definition | See also |
|---|---|---|
| **Command ack** | The synchronous acknowledgment returned to the frontend after dispatching a `ChatCommand`. Carries accepted/rejected status and, for WebSocket, a correlation ID. | [connection-protocol.md](connection-protocol.md) |
| **Command queue** | The frontend command buffer (`CommandQueue`) that holds commands issued before the WebSocket connection is established, then flushes FIFO on connect. Pending commands are rejected on disconnect or close. | [connection-protocol.md](connection-protocol.md) |
| **Command registry** | The frontend registry mapping command IDs to handler functions, used by `ExtensionManager` to dispatch commands invoked by the UI or extensions. | [extension-system.md](extension-system.md) |
| **Context key** | A shell-side expression key used in extension `when` conditions and keybindings. Built-in keys: `connection`, `chatState`, `chatId`, `sidebarVisible`, `inspectorVisible`. Supports `\|\|`, `&&`, `==`, `!=`, `!` operators. | [extension-system.md](extension-system.md) |
| **Event log store** | The frontend in-memory event ledger (`eventLogStore`). Keeps the raw event array and indexes: `turnMap`, `itemMap`, `requestMap`. Deduplicates by `seq`; truncates at 10,000 events. Supports pattern queries (`foo.*`, `*`). | [chat-extension.md](chat-extension.md) |
| **Event router** | The frontend routing layer (`EventRouter`) that clones and freezes events before delivery to pattern or predicate subscribers. Catches handler exceptions so one extension cannot break routing for others. | [extension-system.md](extension-system.md) |
| **Extension capability** | Capability tier for an extension invocation context: `subprocess` (can launch processes), `kernel` (elevated internal access), `hitl` (can prompt the user). Defaults to denied for CLI and MCP callers. | [../../concepts/extension-system.md](../../concepts/extension-system.md) |
| **Extension command registry** | The backend registry (`ExtensionCommandRegistry`) of all extension command specs. Enforces: no duplicate IDs; non-first-party commands cannot expose CLI or MCP surfaces. Provides surface-aware lookup. | [../../concepts/extension-system.md](../../concepts/extension-system.md) |
| **Extension command spec** | The typed declaration of one operation: name, description, parameters, sync handler, surface membership (CLI/MCP/HTTP), first-party flag, app-server requirement, and required capabilities. Declared via `ExtensionCommandSpec.from_op()`. | [../../concepts/extension-system.md](../../concepts/extension-system.md) |
| **Extension context** | The per-extension runtime object (`ExtensionContextImpl`) exposing shell APIs: `connection`, `events`, `commands`, `views`, `statusBar`, `storage`, `state`, `contextKeys`, `logger`. | [extension-system.md](extension-system.md) |
| **Extension dispatcher** | The backend routing layer that takes an extension invocation through: spec lookup → first-party gate → surface gate → app-server gate → args validation → capability validation → handler call → result normalization. | [../../concepts/extension-system.md](../../concepts/extension-system.md) |
| **Extension invocation context** | The per-call runtime context for extension dispatch. Fields: `caller_surface`, `project_uuid`, `work_id`, `work_path`, `spawn_id`, `capabilities`, `request_id`. | [../../concepts/extension-system.md](../../concepts/extension-system.md) |
| **Extension manager** | The frontend runtime lifecycle engine (`ExtensionManager`). Registers extensions, adds manifest contributions to layout state, activates extensions on demand (immediate, visible, event, command), tracks runtime state and errors. | [extension-system.md](extension-system.md) |
| **Extension manifest** | The frontend manifest (`ExtensionManifest`) declaring an extension's identity, version, required shell API version, activation policy, trust model, and contributions (views, commands, rail items, status bar items, keybindings). | [extension-system.md](extension-system.md) |
| **Extension registry** | The frontend registry (`ExtensionRegistry`) of loaded extensions. Enforces: no duplicate IDs, API version compatibility check, topological dependency ordering. | [extension-system.md](extension-system.md) |
| **First-party** | The extension flag marking Meridian-internal commands (`first_party=True`). Only first-party extensions may be exposed on CLI or MCP surfaces. Third-party extensions are restricted to HTTP surface. | [../../concepts/extension-system.md](../../concepts/extension-system.md) |
| **Layout manager** | The frontend visual composition layer (`LayoutManager`). Reads registered views, rail items, and status bar items from `layoutStore`. Determines mobile mode, renders responsive panels, and persists layout. | [overview.md](overview.md) |
| **Layout store** | The frontend Zustand store (`layoutStore`) tracking registered views, rail items, and status bar items contributed by extensions. View registration is extension-scoped; removing an extension removes all its contributions. | [extension-system.md](extension-system.md) |
| **Portless launcher** | The dev frontend launcher (`PortlessLauncher`) that runs Vite through portless for stable HTTPS routes (`https://<service>.meridian.localhost`). Supports Tailscale and funnel sharing. | [../../architecture/chat/dev-frontend.md](../../architecture/chat/dev-frontend.md) |
| **Raw Vite launcher** | The dev frontend launcher (`RawViteLauncher`) that runs Vite directly on a free localhost port with the backend proxied. Used when portless is not available or disabled. | [../../architecture/chat/dev-frontend.md](../../architecture/chat/dev-frontend.md) |
| **Shell provider** | The React root component (`ShellProvider`) orchestrating chat bootstrap, runtime service creation (WebSocket, EventRouter, CommandRegistry, ExtensionManager), event routing, and startup. | [overview.md](overview.md) |
| **Shell store** | The frontend Zustand store (`shellStore`) for connection state, chat ID, chat state, backend capabilities, layout state, active extension ID, and extension runtime states. Layout substate is persisted; extension state is volatile. | [overview.md](overview.md) |
| **Surface** | Which external interfaces an extension command is accessible from. Three surfaces: CLI, MCP, HTTP. Declared per-extension. Third-party extensions are restricted to HTTP surface. | [../../concepts/extension-system.md](../../concepts/extension-system.md) |
| **Token batcher** | The frontend buffer (`TokenBatcher`) for `content.delta` events. Accumulates deltas and flushes on `requestAnimationFrame`. Non-delta events pass through immediately. `flushNow()` drains on disconnect or close. | [connection-protocol.md](connection-protocol.md) |
| **WebSocket client** | The frontend reconnecting WebSocket transport (`WebSocketClient`). Builds `ws[s]://host/ws/chat/{chatId}?last_seq=...` URLs using `partysocket`. Tracks connection state in `shellStore`. Buffers `content.delta` events through `TokenBatcher`. | [connection-protocol.md](connection-protocol.md) |
| **WebSocket fanout** | The per-chat WebSocket broadcast layer (`WebSocketFanOut`). Maintains bounded per-client queues; drops slow clients with close code `1008` plus a `backpressure:reconnect_with_last_seq` message. | [../../architecture/chat/event-pipeline.md](../../architecture/chat/event-pipeline.md) |
