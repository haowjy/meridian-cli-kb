# Codebase Vocabulary

Terms covering harness infrastructure, hooks and plugins, safety and permissions, the launch system, state management, config loading, and platform abstractions. For the full vocabulary index see [../vocabulary.md](../vocabulary.md).

| Term | Definition | See also |
|---|---|---|
| **Artifact** | A file produced by or for a spawn, stored under `~/.meridian/projects/<uuid>/spawns/<id>/`. Canonical artifacts: `prompt.md`, `report.md`, `history.jsonl`, `stderr.log`, `params.json`, `tokens.json`, `heartbeat`. | [../concepts/spawn-lifecycle.md](../concepts/spawn-lifecycle.md) |
| **Artifact store** | The protocol for artifact persistence, implemented as `LocalStore` (file-backed) and `InMemoryStore` (for tests). | [../architecture/state-system.md](../architecture/state-system.md) |
| **Bootstrap doc** | A markdown document loaded into primary session prompts at launch. Discovered from `.mars/skills/<name>/resources/BOOTSTRAP.md` (tier 1) and `.mars/bootstrap/<name>/BOOTSTRAP.md` (tier 2), loaded in deterministic order. | [../concepts/bootstrap-docs.md](../concepts/bootstrap-docs.md) |
| **Agent copy (Claude)** | A Mars `[settings.meridian.agent_copy]` configuration that materializes Meridian-authored agent profiles into Claude's native agent surface (`.claude/agents/`). When `harnesses = ["claude"]` and `.claude` is a target, Meridian allows Claude's generic `Agent` tool because the native surface is Meridian-owned. | [../concepts/harness-abstraction.md](../concepts/harness-abstraction.md#claude-native-agent-routing) |
| **Budget** | An execution cost limit with `per_run_usd` and `per_space_usd` fields. Enforced at runtime by `LiveBudgetTracker`. | [architecture/sandbox-projection.md](../architecture/sandbox-projection.md) |
| **Builtin hook** | An in-process hook implementation (not a shell command) exposing `name`, `requirements`, `default_events`, `default_interval`, and `execute()`. Currently: `git-autosync`. | [../concepts/hooks-and-plugins.md](../concepts/hooks-and-plugins.md) |
| **Composition surface** | Alias for **Launch surface**. See that entry. | [../architecture/launch-system.md](../architecture/launch-system.md) |
| **Context** | Disambiguation: (a) **filesystem context** — resolved paths to work directory, KB, and archive; (b) **launch context** — the `LaunchContext` dataclass produced by the composition factory. Disambiguated by surrounding usage. | [../concepts/context-resolution.md](../concepts/context-resolution.md) |
| **Continue replay contract** | Launch-owned normalized contract for exact continue. `continue_replay.py` builds `ContinueReplayContract` from a resolved source reference plus optional `LaunchPolicySnapshot`; primary and spawn continue both consume it instead of independently re-deriving identity, work, task cwd, passthrough args, or session metadata. | [../concepts/session-initiation.md](../concepts/session-initiation.md) |
| **Claude built-in agent denial** | Unconditional default denial of Claude's bundled `Explore`, `Plan`, `General-purpose`, and `general-purpose` native agents. Injected into `--disallowedTools` by `project_claude.py`. No config toggle — a Meridian platform policy. | [../concepts/harness-abstraction.md](../concepts/harness-abstraction.md#claude-native-agent-routing) |
| **Context source type** | Enum for how a context path is resolved: `local` (resolved relative to project root or absolute) or `git` (resolved relative to a managed git clone of the configured remote). | [../concepts/context-resolution.md](../concepts/context-resolution.md) |
| **Direct adapter** | An in-process harness adapter that executes prompts via a Python API call rather than launching a subprocess. Used in tests and local execution. Implements `InProcessHarness` rather than `SubprocessHarness`. | [../concepts/harness-abstraction.md](../concepts/harness-abstraction.md) |
| **Delegation preference guidance** | System-prompt text rendered by Mars into the `prompt_surface.inventory_prompt` bundle field (adjacent to the `# Meridian Agents` list). Tells Claude-hosted primary sessions to use `meridian spawn` for subagent delegation and reserve native `Agent` only for explicit Claude-native/model-specific cases. Meridian embeds the Mars-rendered string verbatim — no Python-side guidance renderer. | [../concepts/composition-pipeline.md](../concepts/composition-pipeline.md#agent-inventory-prompt) |
| **Driving adapter** | One of the four top-level entry paths that construct a `SpawnRequest` and `LaunchRuntime`, then call `build_launch_context()`. The four: primary CLI path, spawn subprocess path, REST app path, CLI streaming-serve path. None re-implement composition logic. | [../architecture/launch-system.md](../architecture/launch-system.md) |
| **EARS** | Easy Approach to Requirements Syntax. A structured format for expressing behavioral requirements used in design packages and plan phases. Each EARS statement is a traceable unit for QA coverage verification. | [../ecosystem/prompt-packages/meridian-dev-workflow.md](../ecosystem/prompt-packages/meridian-dev-workflow.md) |
| **Fork** | A harness capability allowing a new spawn to start from an existing session's state. Materialized via `fork.py:materialize_fork()`, the sole callsite for `adapter.fork_session()`. | [../concepts/harness-abstraction.md](../concepts/harness-abstraction.md) |
| **Harness bundle** | The registration unit for a harness: ties together an adapter class, spec class, extractor class, and connection class. Validated at registration time for duplicate IDs, extractor validity, and connection compatibility. | [../concepts/harness-abstraction.md](../concepts/harness-abstraction.md) |
| **Harness capabilities** | The `HarnessCapabilities` struct declaring what features a harness adapter supports: stream events, stdin prompt, session resume/fork, native skills, native agents, programmatic tools, primary launch, native file injection. | [../concepts/harness-abstraction.md](../concepts/harness-abstraction.md) |
| **Harness connection** | The abstract transport contract for communicating with a running harness. Concrete implementations: `claude_ws` (stdin/stdout stream-json), `codex_ws` (WebSocket JSON-RPC), `opencode_http` (HTTP + SSE). | [harness-adapters.md](harness-adapters.md) |
| **Harness extractor** | The protocol for session ID and report extraction from harness output artifacts. `StreamingExtractor` wraps a concrete extractor and prefers live connection session IDs over artifact-extracted ones. | [harness-adapters.md](harness-adapters.md) |
| **Harness ID** | The stable type identifying a harness (`HarnessId`). Values: `claude`, `codex`, `opencode`, `direct`. Distinct from `TransportId`, which identifies the underlying transport mechanism. | [../concepts/harness-abstraction.md](../concepts/harness-abstraction.md) |
| **Heartbeat** | A file (`spawns/<id>/heartbeat`) touched every 30 seconds by a running harness process. The reaper's primary liveness signal. A heartbeat older than 120 seconds, combined with no live `runner_pid`, indicates an orphaned spawn. | [../concepts/spawn-lifecycle.md](../concepts/spawn-lifecycle.md) |
| **Hook** | A lifecycle event handler that fires at defined moments (spawn start, spawn end, session open, etc.). Can be a builtin (e.g., `git-autosync`) or a user-defined shell command. | [../concepts/hooks-and-plugins.md](../concepts/hooks-and-plugins.md) |
| **Hook context** | The hook invocation payload carrying spawn ID, status, agent, depth, work ID, and other runtime fields. Serialized as `MERIDIAN_*` env vars (`to_env()`) or JSON on stdin (`to_json()`). Schema version 1. | [../concepts/hooks-and-plugins.md](../concepts/hooks-and-plugins.md) |
| **Hook dispatcher** | The lifecycle-driven hook execution engine. Runs after store writes are complete. Dispatches in registry order; observe/post events are fail-open; gate events honor `failure_policy`. | [../concepts/hooks-and-plugins.md](../concepts/hooks-and-plugins.md) |
| **Hook event class** | Categorization of hook lifecycle events controlling defaults and fail behavior. `observe`: 30s timeout, fail-open. `post`: 60s timeout, fail-open. `gate`: 1s timeout, honors `failure_policy` (can block dispatch). | [../concepts/hooks-and-plugins.md](../concepts/hooks-and-plugins.md) |
| **Hook registry** | The ordered registry of enabled hooks (`HookRegistry`). Sorted by source precedence rank, then negative priority, then declaration order. Disabled hooks are excluded from the event index. | [../concepts/hooks-and-plugins.md](../concepts/hooks-and-plugins.md) |
| **Interval tracker** | The hook throttling mechanism (`IntervalTracker`). Throttles hooks to run at most once per configured interval (e.g., `interval = "30m"`). Persists the last-run timestamp atomically per runtime root. | [../concepts/hooks-and-plugins.md](../concepts/hooks-and-plugins.md) |
| **Launch surface** | The `LaunchCompositionSurface` enum value controlling composition behavior. `PRIMARY`: full primary-path composition (session seed, inventory prompt). `SPAWN_PREPARE`: spawn path (prompt assembly from refs, context_from). `DIRECT`: minimal composition (bypass surface, harness from request only). | [../architecture/launch-system.md](../architecture/launch-system.md) |
| **Live budget tracker** | The runtime cost accumulator (`LiveBudgetTracker`). Tracks `run_cost_usd` (monotonically increasing) and `space_spent_usd`. `check()` enforces per-run and per-space limits. | [architecture/sandbox-projection.md](../architecture/sandbox-projection.md) |
| **Managed model override** | A model token Meridian passes to a harness. Absence means Meridian omits `--model` (or the harness equivalent) and the harness chooses its default. Persisted `LaunchPolicySnapshot.model == ""` is legacy JSON for this absence; in-memory replay uses `None`. | [../decisions/model-resolution.md](../decisions/model-resolution.md#model-optional-empty-model) |
| **MCP** | Model Context Protocol. Meridian exposes an MCP stdio server (via FastMCP) with two tools: `extension_list_commands` and `extension_invoke`. These dispatch through the same extension registry as the CLI and REST API. | [../architecture/app-server.md](../architecture/app-server.md) |
| **Meridian config** | The main settings model (`MeridianConfig`) loaded from TOML config files. Sections: `[defaults]`, `[timeouts]`, `[harness]`, `[primary]`, `[output]`, `[state]`. Loaded with source precedence: init args > env > local TOML > project TOML > user TOML. | [../concepts/config-precedence.md](../concepts/config-precedence.md) |
| **Operation runtime** | The per-call resolved context for ops layer operations (`OperationRuntime`). Fields: `project_root`, `config`, `harness_registry`, `artifacts`, `sink`. Built by `resolve_runtime_root_and_config()` and related helpers. | [../architecture/system-overview.md](../architecture/system-overview.md) |
| **Permission config** | The transport-neutral frozen permission configuration (`PermissionConfig`). Holds sandbox mode, approval mode, allowed tools, and disallowed tools. Constructed by the permission pipeline; consumed by harness adapters. | [architecture/sandbox-projection.md](../architecture/sandbox-projection.md) |
| **Plugin API** | The stable v1 Python API exposed to hooks and plugins (`meridian.plugin_api`). Provides typed helpers for state access, config reads, git operations, and filesystem interaction, shielding plugins from internal implementation changes. | [../concepts/hooks-and-plugins.md](../concepts/hooks-and-plugins.md) |
| **Primary event scope** | The harness-native identity that marks which streaming events belong to the parent spawn when the stream can also include child activity. Codex uses the main `threadId`; OpenCode uses the launched parent `sessionID`. Child events remain in history but do not complete, fail, clear signals for, or supply reports for the parent. | [../concepts/harness-abstraction.md](../concepts/harness-abstraction.md#primary-event-scope) |
| **Process launcher** | The abstract process launching interface (`ProcessLauncher`). Concrete implementations: `SubprocessLauncher` (cross-platform subprocess), `PtyLauncher` (Unix PTY for primary session ID detection), `WindowsLauncher`. Selected by `ProcessLauncherSelector`. | [../architecture/launch-system.md](../architecture/launch-system.md) |
| **Project paths** | The project-local `.meridian/` file paths (`ProjectPaths`). Holds the project ID file, workspace config paths, context roots, and project-specific bootstrap locations. Distinct from `RuntimePaths` (user-level). | [../architecture/state-system.md](../architecture/state-system.md) |
| **Projection** | The harness-specific transformation of harness-agnostic semantic IR (`ComposedLaunchContent`) into concrete harness channel assignments (`ProjectedContent`). Performed by each adapter's `project_content()` method. | [../concepts/composition-pipeline.md](../concepts/composition-pipeline.md) |
| **PTY launcher** | The Unix PTY-based process launcher (`PtyLauncher`). Used for primary launches where session ID detection requires reading from a TTY stream. Not used on Windows. | [../architecture/launch-system.md](../architecture/launch-system.md) |
| **Reconciliation** | The process of comparing observed spawn/session state against expected state and repairing discrepancies. Triggered on every read path. Idempotent — re-running after interruption converges to correct state. | [../concepts/state-model.md](../concepts/state-model.md) |
| **Reference item** | A file or directory passed to a spawn via `-f` / `--file`. Expanded at launch time into the prompt (files as full content, directories as tree listings). Template variable substitution (`{{KEY}}`) applies to reference contents. | [../concepts/composition-pipeline.md](../concepts/composition-pipeline.md) |
| **Resolved context** | The canonical immutable runtime context (`ResolvedContext`). Holds spawn ID, depth, project root, runtime root, chat ID, work ID, and resolved context directory paths. Used by launch, ops, and child env composition through one immutable seam. | [../concepts/context-resolution.md](../concepts/context-resolution.md) |
| **Runtime overrides (env)** | `$MERIDIAN_RUNTIME_DIR` (absolute or relative path to override the runtime root entirely) and `$MERIDIAN_HOME` (override the user root default). Used in tests and CI to isolate state. | [../concepts/state-model.md](../concepts/state-model.md) |
| **Runtime paths** | The user-level directory structure (`RuntimePaths`) under `~/.meridian/projects/<uuid>/`. Contains per-spawn `spawns/<id>/state.json`, `sessions.jsonl`, artifact dirs, hook state, telemetry, and cache. Keyed by project UUID so it survives repo renames. | [../architecture/state-system.md](../architecture/state-system.md) |
| **Sandbox** | The permission/approval layer that controls what tools a harness process may use. Configured via approval mode, sandbox mode, permission resolvers, and budget limits. | [architecture/sandbox-projection.md](../architecture/sandbox-projection.md) |
| **Sandbox mode** | The permission tier applied to a spawn. Values: `default` (harness decides), `read-only`, `workspace-write`, `danger-full-access`. Mapped to harness-specific permission flags during launch. | [architecture/sandbox-projection.md](../architecture/sandbox-projection.md) |
| **Secret spec** | A frozen Pydantic model (`SecretSpec`) holding a secret key and value for redaction. Applied to stdout/stderr streams and artifact payloads. Sorted by length descending before substitution so overlapping secrets redact safely. | [architecture/sandbox-projection.md](../architecture/sandbox-projection.md) |
| **Session record** | The materialized session state (`SessionRecord`) produced by replaying session JSONL events. Holds session ID, start time, status, PID, and harness session ID. | [../architecture/state-system.md](../architecture/state-system.md) |
| **Streaming extractor** | The extractor wrapper (`StreamingExtractor`) that prefers live connection session IDs and falls back to artifact/session detection. Wraps the concrete per-harness extractor. | [harness-adapters.md](harness-adapters.md) |
| **Template variable** | A `{{KEY}}` placeholder in prompt text or reference file contents, replaced at launch time via `--prompt-var KEY=VALUE`. Keeps variable content visible in prompt logs and dry-run output. | [../concepts/composition-pipeline.md](../concepts/composition-pipeline.md) |
